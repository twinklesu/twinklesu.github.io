# 02. 변수와 자료형, 연산자


## val 과 var

**val**은 변경이 되지 않는 변수, **var**은 변경할 수 있는 변수
우선적으로 **val**변수를 선언한 후, 변경할 일이 생긴다면 **var**로 변경하기

코틀린은 자료형을 추론하기 때문에 datatype 선언을 하지 않아도 되지만, 그런 경우 초기화 시 값할당을 **무조건**해줘야 한다.

## String 선언

문자열의 경우 스택에 참조 주소가 저장되고, 힙에 값이 저장된다.
이 때 같은 문자열을 선언해 준 경우 하나의 힙 영역을 공유한다. 

## null

코틀린의 모든 자료형은 **null**을 허용하지 않는다. 
**null**할당을 위해서는 ```kotlin var str: String? = null```과 같이 변수타입 뒤에 **?**를 붙여줘야한다.

 - 세이프 콜 (**?.**)
  - `println("str: ${str?.length}")` str에 대해 null 검사 후, null 일 시 null 출력, 아닐시 length 접근
 - non-null 단정기호 (**!!.**)
  - `str!!.length` str이 null이 절대 아니라고 생각하기 때문에, 실행 시 null인 경우 **NullPointerExcetion**발생
- 조건문 활용
  - `val len = if(str!=null) str.length else -1`
- Elvis 연산자 (**?:**)
  - `str?.length ?: -1` not null 이면 전자, null이면 후자 실행

## 비교연산자

- `==`는 값 비교
- `===`는 참조 주소 비교(값과 무관)

**기본형** 자료형의 경우, stack에 변수 값만 저장되어 있기 때문에, `==`과 `===`이 그 값을 비교한다. 
하지만, **참조형** 자료형의 경우, `==`는 주소를 타고 이동해 heap의 값을 비교하지만, `===`는 heap 영역을 가르키는 주소 자체를 비교한다. 

- 코틀린에서는 참조형으로 **-128~127**의 값을 선언하면, 값을 **캐시**에 저장하고 캐시를 참조한다.
